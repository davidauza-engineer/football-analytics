We'd like to highlight a subtle aspect of data types. In Python, when you multiply, add, or subtract two integer numbers, the result remains an integer. However, when you divide an integer by another integer, the result is always a floating-point number. Consider this example:

print(type(4 + 3))
print(type(4 - 3))
print(type(4 * 3))
print(type(4 / 3))

The output:

<class 'int'>
<class 'int'>
<class 'int'>
<class 'float'>

This might not come as a surprise, as it's natural to expect a floating-point number when dividing non-divisible numbers. However, it's worth noting that the result is still a float even when dividing two numbers that divide evenly, like this:

print(type(4 / 2))

The output remains <class 'float'>.

print(12 / 3)
